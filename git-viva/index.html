Git

Fluxo básico e padrões

Inicializando um repositório

Repositório local

git init

Cria um repositório local

O repositório estará vazio

O repositório não estará associado a um repositório remoto

O repositório terá as configurações padrão do seu git

git remote add origin http://caminhodorepositorio.git

Associa o repositório local criado com um repositório remoto escolhido

Repositório remoto

git clone http://caminhodorepositorio.git

Cria um repositório local

O repositório local será igual ao repositório remoto de origem

OBS: Para clonar um repositório alterando nome da pasta utilize

git clone http://caminhodorepositorio.git nomedapasta

Configuração de usuário

git config user.name “Nome do Usuário”

Define o nome do usuário que será utilizado nos commits do repositório ativo

git config user.email usuario@email.com

Define o email do usuário que será utilizado nos commits do repositório ativo

OBS: Para definir o mesmo nome e email de usuário para todos os respositórios utilize o parâmetro --global

git config --global user.name “Nome do Usuário”

git config --global user.email usuario@email.com

Fluxo básico

Diretório de trabalho
(working directory)

Snapshot dos arquivos atuais

Identifica os arquivos incluídos, modificados ou excluídos

Área de preparação
(staging area)

Armazena as alterações realizadas

Prepara os arquivos para serem commitados

Repositório local
(local repository)

Versão local do último commit

Gravação das alterações armazenadas na máquina local

Repositório remoto
(remote repository)

Versão remota dos repositórios locais

Compartilhado com os membros do projeto

Working directory => Staging area


git add caminho/do/arquivo

Salva um arquivo adicionado ou alterado


git rm caminho/do/arquivo

Salva um arquivo removido

Working directory => Staging area

git add .

Salva todos os arquivos adicionados ou alterados

git rm .

Salva todos os arquivos removios

git add -A

Salva tudo

Staging area => Local repository

git commit

Cria uma versão do trabalho com as alterações salvas na área de praparação

Gera uma hash para referenciar a HEAD da branch

Possui cabeçalho para resumir o que foi feito em até 50 caracteres, e corpo para dar mais detalhes sobre o que foi citado no cabeçalho

Staging area => Local repository

git commit -m "Cabecalho do commit"

Permite criar um commit sem abrir o prompt, digitando somente o cabeçalho na mesma linha

git commit -a

Salva automaticamente os arquivos alterados e deletados, mas não os adicionados

Local repository => Remote repository

git push

Envia os commits da branch local ativa para a branch remota padrão definida

Local repository => Remote repository

git push repository branch

Envia os commits da branch local ativa para a branch remota especificada

Local repository => Remote repository

repository

  • Nome do repositório remoto definido.
    Ex: git push origin branch

  • Caminho do repositório remoto definido.
    Ex: git push http://caminhodorepositorio.git branch

    • Remote repository => Local repository

    git fetch

    Atualiza as referências das branches locais em relação às remotas

    Remote repository => Local repository

    git pull

    Atualiza a referência da branch local ativa em relação à remota padrão e faz a mesclagem automaticamente

    git pull repository branch

    Atualiza a referência da branch local ativa em relação à remota especificada e faz a mesclagem automaticamente

    Obs: git pull = git fetch + git merge

    Comandos auxiliares

    git status

    Mostra o estado atual dos arquivos da branch, e a situação da branch local em relação à remota

    git branch

    Lista as branches existentes no repositório remoto

    git branch nomeDaBranch

    Cria uma nova branch local

    git checkout nomeDaBranch

    Muda a branch ativa

    git checkout -b nomeDaBranch

    Cria uma nova branch local e faz dela a branch ativa automaticamente

    git merge nomeDaBranch

    Mescla a branch referida na branch ativa

    git branch -d nomeDaBranch

    Deleta a branch referida

    git reset

    Desfaz o último commit da branch e retorna as alterações para o estado de não salvas. Pode ser usado o índice do array de commits ou a hash de um commit específico como referência

    git commit --amend

    Permite alterar as informações do último commit

    git log

    Mostra o log de commits

    git diff

    Discrimina as alterações não salvas realizadas nos arquivos

    git stash

    Salva as alterações realizadas nos arquivos, e retorna a referência da HEAD para o último commit

    git stash pop

    Recupera as informações salvas no array de stashes

    git checkout caminho/do/arquivo

    Desfaz as alterações realizadas no arquivo especificado, retornando o mesmo para o estado do último commit

    Resolvendo conflitos

    git pull ou git merge

    Correções nos arquivos

    git add e/ou git rm
    git commit
    git push

    Obs: Pode ser necessário usar git push -f para forçar um push que o git considera como uma alteração diferente do que está na branch remota
